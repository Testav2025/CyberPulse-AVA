Refactor CyberPulse AVA from a security operations dashboard into an employee-first security companion platform.

The target audience is:

- Frontline manufacturing workers
- Office staff
- HR
- Finance
- Operations
- Managers

The platform must use plain English and avoid technical security jargon.

GENERAL PRINCIPLES

- Every page should answer:
  "Am I safe?"
  "Do I need to do anything?"
  "How can I improve my security?"

- Remove or hide highly technical information.
- Introduce role-based experiences.
- Make the UI feel friendly, modern, supportive and educational.
- Use Microsoft Fluent design principles.
- Prioritise accessibility, simple language and mobile use.

------------------------------------

RENAME SECTIONS

Current -> New

CyberScore -> My Security Score

Darktrace -> Email Protection

Pentera -> Security Improvements

Alerts -> Important Security Messages

Devices -> My Devices

Training -> Cyber Learning

AI Assistant -> Ask AVA

------------------------------------

REMOVE TECHNICAL TERMS

Hide from standard user view:

- CVSS
- ATT&CK
- Exploit Path
- Vulnerability Severity
- Threat Hunting
- Lateral Movement

Translate into plain English.

Example:

"High Priority Security Improvement"

instead of

"CVSS 8.9 Vulnerability"

------------------------------------

EXPAND ASK AVA

Ask AVA should become the centrepiece of the platform.

Prebuilt prompts:

- Am I secure?
- Why did my score change?
- Have I received suspicious emails?
- What should I improve this week?
- What is phishing?
- How do I stay safe online?
- Has my device got any issues?

Responses should use simple non-technical language.
